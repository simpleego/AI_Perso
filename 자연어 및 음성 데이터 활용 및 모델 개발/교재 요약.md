## [cite_start]4. 자연어 데이터의 고유한 특성 [cite: 4, 5]

* **불규칙성**: 문법 규칙의 예외 (예: 먹다 -> 먹는다/먹었습니다)
* **모호성**: 하나의 표현이 여러 의미 (예: "배" -> 선박/과일/신체)
* **다의성**: 문맥에 따른 의미 변화 (예: "은행" -> 금융기관/식물)
* **문맥 의존성**: 주변 상황에 따른 의미 차이
* [cite_start]**언어별 차이**: 한국어(교착어) vs 영어(어순 중심) [cite: 7]

## [cite_start]5. 텍스트 전처리 (Preprocessing) [cite: 9]

### 전처리 워크플로우
1. [cite_start]**Raw 텍스트**: 정제되지 않은 데이터 (뉴스, SNS 등) [cite: 10]
2. [cite_start]**정규화**: 소문자화, 특수문자 제거, 공백 정규화 [cite: 11]
3. [cite_start]**불용어 제거**: 분석에 불필요한 단어 제거 (is, the, 은, 는 등) [cite: 11, 12]
4. [cite_start]**어간/표제어 추출**: 단어를 기본형으로 변환 [cite: 11, 13]

### [cite_start][실습] 한국어 전처리 파이프라인 [cite: 17]
```python
import re
from konlpy.tag import Okt

def preprocess_korean_text(text):
    okt = Okt()
    # 소문자화 및 특수문자 제거
    text = re.sub(r"[^\w\s]", " ", text.lower())
    # 형태소 분석
    tokens = okt.morphs(text)
    # 불용어 제거
    stopwords = {"은", "는", "이", "가", "을", "를", "에", "의"}
    return [t for t in tokens if t not in stopwords and t.strip()]

sample = "오늘은 날씨가 좋아서 공원에서 산책을 했습니다."
print(preprocess_korean_text(sample))
```

## [cite_start]6. 토큰화 및 임베딩 [cite: 19, 34]

### 토큰화 방식
* [cite_start]**단어 단위**: 공백 기준 분할 (영어에 효과적) [cite: 20]
* [cite_start]**형태소 단위**: 의미 최소 단위 분리 (한국어 필수 - KoNLPy 활용) [cite: 19, 22]
* [cite_start]**하위 단어(Subword)**: BPE, WordPiece 등 최신 LLM 표준 방식 [cite: 26, 31]

### [cite_start]임베딩 기술의 발전 [cite: 35]
1. [cite_start]**원-핫 인코딩**: 단순 벡터화, 유사성 표현 불가 [cite: 36]
2. [cite_start]**통계 기반 (TF-IDF)**: 단어 빈도와 희소성 반영 [cite: 39]
3. [cite_start]**예측 기반 (Word2Vec)**: 주변 단어 예측을 통한 학습 [cite: 35]
4. [cite_start]**문맥 기반 (BERT, GPT)**: Transformer 구조를 이용한 양방향 문맥 학습 [cite: 35]
