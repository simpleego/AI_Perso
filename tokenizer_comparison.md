# BPE, WordPiece, SentencePiece 알고리즘 정리

---

## 📊 알고리즘 비교 개요

| 항목 | BPE | WordPiece | SentencePiece |
|------|-----|-----------|---------------|
| **병합 기준** | 인접 쌍 **빈도** 최대화 | 언어모델 **가능도** 최대화 | **유니그램 LM** 확률 기반 제거 |
| **방향** | Bottom-up (글자→단어) | Bottom-up (글자→단어) | Top-down (전체→서브워드) |
| **공백 처리** | `</w>` 단어 끝 표시 | `##` 이전 토큰 연결 | `▁` 단어 시작 표시 |
| **사전 토크나이즈** | 공백 기준 분리 필요 | 공백 기준 분리 필요 | 불필요 (원시 텍스트 직접 처리) |
| **언어 독립성** | 낮음 (공백 의존) | 낮음 (공백 의존) | 높음 (한국어·일어 등 유리) |
| **한국어 처리** | 형태소 분석 추가 필요 | 형태소 분석 추가 필요 | 공백 없어도 학습 가능 |
| **대표 모델** | GPT-2, RoBERTa, CodeLlama | BERT, DistilBERT, mBERT | T5, LLaMA, Claude, Gemma |
| **어휘 크기 제어** | 병합 횟수로 직접 제어 | 병합 횟수로 직접 제어 | 목표 어휘 크기 설정 후 pruning |
| **복원 가능성** | 완전 복원 가능 | 완전 복원 가능 | 완전 복원 가능 (`▁` 제거) |

---

## 1. BPE (Byte Pair Encoding)

> **"자주 붙어 나오는 글자 쌍을 계속 합치면 의미 있는 단위가 된다"**

### 작동 방식

1. **초기화** — 글자 단위로 분할 + `</w>` 추가
2. **빈도 계산** — 인접 쌍의 등장 빈도 통계
3. **병합 (Merge)** — 최고 빈도 쌍을 하나로 합침
4. **반복** — 목표 어휘 크기에 도달할 때까지 반복

### 예시

```
"lower" 토크나이징 과정:
l o w e r </w>
→ lo w e r </w>   (l+o 병합)
→ low e r </w>    (lo+w 병합)
→ lower </w>      (low+er 병합)
```

### 특징

- 구현이 간단하고 직관적
- 공백을 기준으로 사전 분리 필요
- **사용처**: GPT-2, RoBERTa, CodeLlama

---

## 2. WordPiece

> **"두 조각을 합쳤을 때 언어모델의 예측 가능도가 얼마나 높아지는가"**

### 작동 방식

1. **초기화** — 글자 단위 분할 + `##` 접두사 부착
2. **가능도 계산** — 언어모델 기반 가능도(likelihood) 추정
3. **병합 (Merge)** — 가능도가 가장 높아지는 쌍 선택
4. **반복** — 목표 어휘 크기에 도달할 때까지 반복

### 예시

```
"playing" 토크나이징:
p l a y i n g
→ play ##ing
```

> `##`은 앞 토큰에 연결됨을 의미 (단어 중간/끝 조각)

### BPE와의 차이

| | BPE | WordPiece |
|-|-----|-----------|
| 병합 기준 | 빈도(count) | 가능도(likelihood) |
| 결과 품질 | 통계적 | 언어적으로 더 의미 있음 |

### 특징

- 언어적으로 더 의미 있는 서브워드 생성 경향
- **사용처**: BERT, DistilBERT, mBERT

---

## 3. SentencePiece

> **"공백이 없는 언어도 원시 텍스트 그대로 처리한다"**

### 작동 방식

1. **원시 텍스트** — 공백 포함 문자열을 그대로 입력
2. **유니그램 LM** — 확률적 서브워드 후보 선택
3. **어휘 pruning** — 가능도가 낮은 토큰 제거
4. **`▁` 로 공백 표시** — 언어 독립적 처리

### 예시

```
"Hello world" 토크나이징:
▁Hello ▁world

"안녕하세요" 토크나이징 (공백 없어도 가능):
▁안녕 하세요
```

> `▁`는 단어 시작(공백 위치)을 표시

### BPE/WordPiece와의 근본적 차이

- BPE·WordPiece는 공백 기준으로 사전 분리 후 처리 → **공백 없는 언어에 불리**
- SentencePiece는 원시 바이트 수준에서 직접 처리 → **언어 독립적**

### 특징

- 한국어·일본어·중국어 등 공백 없는 언어에 유리
- 별도 형태소 분석기 불필요
- **사용처**: T5, LLaMA, **Claude**, Gemma

---

## 🔑 핵심 요약

```
BPE          → 빈도 기반 병합 (Bottom-up)  | GPT 계열
WordPiece    → 가능도 기반 병합 (Bottom-up) | BERT 계열
SentencePiece→ 확률 기반 제거 (Top-down)   | 최신 LLM (Claude 포함)
```

### 한국어 처리 관점

한국어는 공백이 없거나 형태소가 복잡하기 때문에, **SentencePiece**가 가장 적합합니다. BPE와 WordPiece는 공백 기반 사전 토크나이즈가 필요해 한국어 처리 시 별도 형태소 분석기를 추가해야 합니다.
