## 자연어 데이터의 고유한 특성

- **불규칙성 (Irregularity)**: 문법 규칙의 예외가 많음 (예: “먹다 → 먹는다 / 먹었습니다 / 먹겠네”)
- **모호성 (Ambiguity)**: 하나의 표현이 여러 의미 가능 (예: “비” → boat / pear / stomach)
- **다의성 (Polysemy)**: 같은 단어가 문맥에 따라 다른 의미 (예: “은행” → 금융기관 / 식물)
- **문맥 의존성 (Context-dependence)**: 주변 단어·상황에 따라 의미 달라짐 (예: “나는 오늘도 달린다” → 운동 vs 주식 투자)
- **언어별 차이 (Language-specific traits)**: 언어마다 고유한 문법 구조 존재 (한국어: 교착어 / 영어: 어순 중심 / 중국어: 성조)

### 다의어(Polysemy)와 동음이의어(Homonymy)

| 다의어 (Polysemy) | 동음이의어 (Homonymy) |
| --- | --- |
| 하나의 단어가 여러 관련된 의미를 가지는 경우 | 발음은 같지만 의미가 다른 단어 |
| 예: "머리" - 신체 부위, 사고 능력, 상단 부분 | 예: "배" - 과일, 선박, 복부 |

- 같은 단어라도 문맥에 따라 의미가 달라짐 → 단순 단어 기반 모델은 혼동
- 전통적 통계 기반 모델(N-gram, TF-IDF 등)의 한계
- 현대 NLP (Word Embedding, Transformer, LLM): 문맥 기반 의미 파악, 다의어·동음이의어 구분 가능

### 문맥 의존성 (Context Dependency)

- 단어나 문장의 의미는 주변 문맥에 따라 달라짐
- 문제점: 단순 Bag-of-Words, TF-IDF, Word2Vec은 문맥을 고려하지 못함
- 해결 방향: Contextual Embeddings (ELMo, BERT, GPT), Transformer 구조(Self-Attention)

### 한국어와 영어의 주요 차이점

| 한국어의 특징 | 영어의 특징 |
| --- | --- |
| 교착어: 조사와 어미가 단어에 붙어 문법적 기능 수행 | 굴절어: 단어 자체의 변형으로 문법 표현 |
| 주어-목적어-동사(SOV) 어순: "나는 밥을 먹었다" | SVO 어순: "I ate food" |
| 생략이 빈번: 주어나 목적어가 자주 생략됨 | 주어 필수: 생략 거의 없음 |
| 높임법: 복잡한 존댓말 체계 | 단순한 존댓말 체계 |
| 형태소 분석 필요: 의미 단위 분석 필수 | 공백 기반 토큰화 가능 |

### 실습: 다의어 "배" 의미 분석하기

```python
# 다의어 "배"의 의미 분석하기
sentences = [
    "저는 어제 배를 먹었습니다.",   # 과일
    "우리는 배를 타고 섬으로 갔습니다.", # 선박
    "운동 후에 배가 아랫습니다."     # 신체
]

from konlpy.tag import Okt
okt = Okt()

for i, sentence in enumerate(sentences):
    print(f"\n문장 {i+1}: {sentence}")
    print("형태소 분석 결과:")
    print(okt.pos(sentence))

    words = okt.morphs(sentence)
    if "배" in words:
        idx = words.index("배")
        context = words[max(0, idx-2):min(len(words), idx+3)]
        print(f"'배' 주변 문맥: {context}")
```

## 전처리 과정 (Preprocessing)

- 자연어 데이터 → 그대로는 복잡하고 불규칙적
- 전처리 = Raw 텍스트 → 모델이 이해할 수 있는 형태 변환
- 효율적 분석 & 학습 성능 향상에 필수

### 텍스트 정규화 (Text Normalization)

- 다양한 형태의 텍스트 → 일관된 형식으로 변환
- 주요 작업: 대소문자 통일, 특수문자 제거, 숫자 처리, 공백 정규화, 철자 오류 수정, 축약형 확장

### 불용어 제거 (Stopwords Removal)

불용어: 문장에서 자주 등장하지만 핵심 의미에 기여도가 낮은 단어

| 영어 불용어 예시 | 한국어 불용어 예시 |
| --- | --- |
| "the", "is", "at", "which", "on", "a", "an", "and", "in", "of", "to", "for", "with" 등 | "이", "그", "저", "것", "등", "및", "에서", "에게", "으로", "하다", "있다", "되다" 등 |

```python
# NLTK를 이용한 영어 불용어 제거
from nltk.corpus import stopwords
from nltk.tokenize import word_tokenize

text = "The quick brown fox jumps over the lazy dog"
stop_words = set(stopwords.words('english'))
tokens = word_tokenize(text.lower())
filtered = [w for w in tokens if w not in stop_words]
print(filtered)  # ['quick', 'brown', 'fox', 'jumps', 'lazy', 'dog']

# 한국어 불용어 제거 예시
from konlpy.tag import Okt
okt = Okt()
text = "나는 오늘 학교에 갔습니다"
korean_stopwords = ["나는", "은", "는", "이", "가", "을", "를", "에", "에서"]
tokens = okt.morphs(text)
filtered = [w for w in tokens if w not in korean_stopwords]
print(filtered)  # ['오늘', '학교', '갔습니다']
```

### 어간 추출(Stemming) vs 표제어 추출(Lemmatization)

| 어간 추출 (Stemming) | 표제어 추출 (Lemmatization) |
| --- | --- |
| 규칙 기반, 빠른 처리 속도 | 문맥 고려 → 올바른 기본형(lemma) 추출 |
| 단점: 결과가 어색하거나 불완전할 수 있음 | 단점: 속도 느림, 자원 많이 필요 |

```python
from nltk.stem import PorterStemmer
ps = PorterStemmer()
words = ["running", "runs", "ran", "easily", "fairly"]
print([ps.stem(w) for w in words])  # ['run', 'run', 'ran', 'easili', 'fairli']

from nltk.stem import WordNetLemmatizer
lemmatizer = WordNetLemmatizer()
words = ["running", "runs", "ran", "mice", "better"]
print([lemmatizer.lemmatize(w) for w in words])  # ['running', 'run', 'ran', 'mouse', 'better']
```

### 한국어 전처리 특성

- 한국어는 교착어 → 조사·어미가 단어에 결합 → 형태소 단위 분리 필수
- 조사 처리: "학교에서" → 학교 + 에서
- 어미 처리: "먹었다" → 먹 + 었다
- 복합어 처리: "서울대학교" → 서울 + 대학교
- 띄어쓰기 오류 처리: "나는학교에갔다" → "나는 학교에 갔다"

### 실습: 텍스트 전처리 파이프라인

```python
import re
from konlpy.tag import Okt

# 예시 텍스트
sample_text = "오늘은 날씨가 좋아서 공원에서 산책을 했습니다. 사람들 200 명이 벚꽃을 구경했어요!"

okt = Okt()

RE_PUNCT = re.compile(r"[^\w\s]", flags=re.UNICODE)
RE_NUM = re.compile(r"\d+")

def preprocess_korean_text(text, keep_num=False, num_token=" <NUM>"):
    text = text.lower()  # 소문자화
    text = RE_PUNCT.sub(" ", text)  # 특수문자/구두점 처리
    if keep_num:
        text = RE_NUM.sub(num_token, text)  # 숫자 처리
    else:
        text = RE_NUM.sub(" ", text)
    
    tokens = okt.morphs(text)  # 형태소 분석 (토큰화)
    stopwords = {"은", "는", "이", "가", "을", "를", "에", "의", "으로", "도", "에서"}
    tokens = [t for t in tokens if t not in stopwords and t.strip()]
    return tokens

# 전처리 적용
processed = preprocess_korean_text(sample_text, keep_num=True)
print(processed)
```

## 토큰화 (Tokenization)

- 텍스트 → 의미 있는 최소 단위(토큰)로 분할
- 언어/과제에 따라 단위 정의가 달라짐

| 방식 | 설명 | 예시 |
| --- | --- | --- |
| 단어 단위 (Word-level) | 문장을 단어로 분할 | "I love NLP" → ["I", "love", "NLP"] |
| 형태소 단위 (Morpheme-level) | 의미 최소 단위로 분리 (한국어 중요) | "나는 학교에 간다" → ["나", "는", "학교", "에", "가", "ㄴ다"] |
| 문자 단위 (Character-level) | 개별 글자 단위 | "Hello" → ["H", "e", "l", "l", "o"] |
| 하위 단어 (Subword-level) | 단어보다 작은 단위, 희귀어 처리 강점 | "unhappy" → ["un", "happy"] |

### 단어 단위 토큰화 (Word Tokenization)

```python
# NLTK의 word_tokenize 사용
```

### 한국어 형태소 분석

KoNLPy는 다양한 형태소 분석기를 제공:

| 분석기 | 특징 |
| --- | --- |
| Okt (Open Korean Text) | 신조어 및 줄임말 처리에 강함, 처리 속도 다소 느림 |
| Mecab | C++ 기반 매우 빠른 처리, 설치 복잡 |
| Komoran | 사용자 사전 확장 용이, 처리 속도 다소 느림 |
| Kkma | 매우 정확한 분석, 풍부한 품사 태그, 처리 속도 매우 느림 |

### 형태소 분석기 비교

```python
from konlpy.tag import Okt, Komoran, Kkma
import time

text = "자연어처리는 인공지능의 중요한 분야 중 하나입니다."

okt = Okt()
komoran = Komoran()
kkma = Kkma()

print("OKT:", okt.morphs(text))
print("Komoran:", komoran.morphs(text))
print("Kkma:", kkma.morphs(text))

print("\nOKT 품사 태깅:", okt.pos(text))
print("Komoran 품사 태깅:", komoran.pos(text))
print("Kkma 품사 태깅:", kkma.pos(text))

# 속도 비교
times = {}
for name, analyzer in [("OKT", okt), ("Komoran", komoran), ("Kkma", kkma)]:
    start = time.time()
    for _ in range(100):
        analyzer.morphs(text)
    times[name] = time.time() - start
print("\n속도 비교 (100회 반복):")
for name, elapsed in times.items():
    print(f"{name}: {elapsed:.4f}초")
```

### 하위 단어 토큰화 (Subword Tokenization)

- 단어를 더 작은 의미 단위로 분할
- 단어 단위 의미 보존 + 문자 단위 OOV 완화
- 최신 언어 모델 (BERT, GPT 등)에서 표준 방식

| 방식 | 설명 | 사용 모델 |
| --- | --- | --- |
| BPE (Byte Pair Encoding) | 빈도 높은 문자 쌍 반복 병합 | GPT |
| WordPiece | BPE 변형, 높은 우도(likelihood) 가진 쌍 병합 | BERT |
| SentencePiece | 언어 독립적, raw 텍스트에서 직접 학습 | 다국어 모델 |
| Unigram LM | 확률 모델 기반, 불필요한 토큰 제거 | XLNet |

### Byte Pair Encoding (BPE) 이해하기

```python
# 간소화된 BPE 과정 시뮬레이션
# 단어 목록: ["low", "lower", "newest", "widest"]
# 1. 문자 단위로 분할
# 2. 빈도수 계산 및 병합 (e+s → es)
# 3. 다음 병합 (l+o → lo)
# 4. 계속 병합 (lo+w → low)
# 최종 어휘: ['l', 'o', 'w', 'e', 'r', 'n', 'i', 'd', 's', 't', 'es', 'lo', 'low']
```

### 실습: BPE 토크나이저 사용하기

```python
# Hugging Face tokenizers 라이브러리 설치
!pip install tokenizers

from tokenizers import Tokenizer
from tokenizers.models import BPE
from tokenizers.trainers import BpeTrainer
from tokenizers.pre_tokenizers import Whitespace

corpus = [
    "안녕하세요 자연어처리를 배우고 있습니다",
    "자연어처리는 인공지능의 중요한 분야입니다",
    "토큰화는 자연어처리의 기본 과정입니다",
    "BPE는 자주 등장하는 문자열을 하나의 토큰으로 만듭니다"
]

tokenizer = Tokenizer(BPE(unk_token="[UNK]"))
trainer = BpeTrainer(special_tokens=["[UNK]", "[CLS]", "[SEP]", "[PAD]", "[MASK]"], vocab_size=1000)
tokenizer.pre_tokenizer = Whitespace()

tokenizer.train_from_iterator(corpus, trainer)

encoded = tokenizer.encode("자연어처리 모델을 학습해봅시다")
print(f"토큰 ID: {encoded.ids}")
print(f"토큰: {encoded.tokens}")

tokenizer.save("bpe_korean.json")

vocab = tokenizer.get_vocab()
print(f"어휘 크기: {len(vocab)}")
```

### Transformer 모델의 토큰화

| 모델 | 토큰화 방식 | 특징 |
| --- | --- | --- |

## 임베딩(Embedding)과 언어모델

- 단어/토큰 → 고정 길이 실수 벡터로 변환
- 의미적으로 유사한 단어는 벡터 공간에서 가까움

| 유형 | 방식 | 특징 |
| --- | --- | --- |
| 전통적 표현 | 원-핫 인코딩, TF-IDF, PMI | 희소 벡터, 의미 정보 없음 |
| 예측 기반 임베딩 | Word2Vec, FastText | 분산 표현, 의미 유사도 포착 |
| 문맥 기반 임베딩 | ELMo, BERT, GPT | 문맥에 따라 벡터 동적으로 변화 |

### 원-핫 인코딩 (One-Hot Encoding)

```python
vocabulary = ["자연어", "처리", "임베딩", "모델", "학습"]
word_to_idx = {word: i for i, word in enumerate(vocabulary)}

def one_hot_encode(word, vocab_size):
    vector = [0] * vocab_size
    vector[word_to_idx[word]] = 1
    return vector

embedding_vector = one_hot_encode("임베딩", len(vocabulary))
print(embedding_vector)  # [0, 0, 1, 0, 0]
```

### 통계 기반 표현: TF-IDF

- TF (Term Frequency): 특정 단어가 문서 내에 등장하는 빈도
- IDF (Inverse Document Frequency): 단어가 corpus 전체에서 얼마나 희소한지
- TF-IDF = TF × IDF

```python
from sklearn.feature_extraction.text import TfidfVectorizer

    "임베딩은 자연어 처리의 중요한 기술입니다"
]

vectorizer = TfidfVectorizer()

print(vectorizer.get_feature_names_out())
print(X.toarray())
```

### 통계 기반 표현의 한계

- 희소성 문제: 대규모 어휘에서 대부분의 벡터가 0
- 의미 관계 포착 불가: "자동차"와 "차량"이 완전히 다른 벡터
- 다의어 처리 불가: "배"의 여러 의미 구분 불가
- 확장성 문제: 새 단어 추가 시 전체 재계산 필요

### Word2Vec: 분산 표현의 핵심

- 2013년 Google Tomas Mikolov 제안
- 분포 가설: "비슷한 문맥에 등장하는 단어는 비슷한 의미를 가진다"

| 학습 방식 | 설명 |
| --- | --- |
| CBOW (Continuous Bag of Words) | 주변 단어들로 중심 단어 예측 |
| Skip-gram | 중심 단어 → 주변 단어 예측 (더 성능 좋음, 계산 비용 높음) |

### Word2Vec의 특징과 장점

- 의미적 유사성 포착: 유사한 의미의 단어들이 벡터 공간에서 가깝게 위치
- 단어 간 관계 표현: "왕 - 남자 + 여자 = 여왕"
- 조밀한 벡터 표현: 100~300 차원
- 전이 학습 가능: 대규모 코퍼스에서 학습한 임베딩 재활용

### 실습: Gensim으로 Word2Vec 학습하기

```python
    print(f"{word}: {similarity:.4f}")

    result = model.wv.most_similar(positive=["딥러닝", "언어"], negative=["컴퓨터"], topn=3)
```

```python
import torch
from scipy.spatial.distance import cosine

tokenizer = BertTokenizer.from_pretrained('bert-base-multilingual-cased')
model = BertModel.from_pretrained('bert-base-multilingual-cased')

sentences = [
    "자연어 처리는 컴퓨터가 인간의 언어를 이해하는 분야입니다.",
    "배를 타고 여행을 떠났습니다.",
    "배가 너무 달아서 많이 먹었습니다."
]

for sentence in sentences:
    inputs = tokenizer(sentence, return_tensors="pt", padding=True, truncation=True)
    with torch.no_grad():
        outputs = model(**inputs)
    sentence_embedding = outputs.last_hidden_state[:, 0, :].numpy()
    token_embeddings = outputs.last_hidden_state.numpy()

# 다의어 "배"의 문맥에 따른 임베딩 비교
def get_word_embedding(sentence, word):
    inputs = tokenizer(sentence, return_tensors="pt")
    with torch.no_grad():
        outputs = model(**inputs)
    tokens = tokenizer.convert_ids_to_tokens(inputs['input_ids'][0])
    try:
        word_idx = tokens.index(word)
        return outputs.last_hidden_state[0, word_idx, :].numpy()
    except ValueError:
        return None

ship_embedding = get_word_embedding("배를 타고 여행을 떠났습니다.", "배")
fruit_embedding = get_word_embedding("배가 너무 달아서 많이 먹었습니다.", "배")

if ship_embedding is not None and fruit_embedding is not None:
    similarity = 1 - cosine(ship_embedding, fruit_embedding)
    print(f"두 '배'의 임베딩 유사도: {similarity:.4f}")
```

## 음성 데이터 활용

### 음성 신호 처리 (Speech Processing)

- 사람의 음성을 디지털 신호로 변환 → 분석 → 변형 → 인식
- 실생활 응용: 스마트 스피커, 자동 자막, 통화 잡음 제거, 감정/화자 분석

### 음성이 담고 있는 정보들

| 정보 유형 | 설명 |
| --- | --- |
| 언어적 정보 (Linguistic) | 단어와 문장의 의미, ASR의 주 대상 |
| 준언어적 정보 (Paralinguistic) | 감정, 억양, 강조, 태도 |
| 화자 정보 (Speaker-related) | 성별, 나이, 발화 습관 |
| 환경 정보 (Environmental) | 배경 잡음, 공간 반향, 채널 특성 |

### 아날로그 → 디지털 파이프라인

1. 마이크 (아날로그)
2. A/D 변환 (샘플링, 양자화)
3. 프레임 분할/윈도잉
4. 특징 추출 (MFCC, 멜스펙트로그램, 에너지, F0)
5. 모델링 (GMM, DNN, RNN, Transformer)
6. 후처리 & 디코딩

### 디지털화의 기초: 양자화와 비트깊이

- **양자화**: 연속적인 진폭 값을 유한한 레벨로 근사
- **비트 깊이**: 각 샘플 표현에 쓰이는 비트 수 (8-bit, 16-bit, 24-bit)
- **파일 포맷**: WAV (무손실), MP3 (손실 압축), AAC

### 음성과 일반 신호의 차이

- **비정상성 (Non-stationarity)**: 시간에 따라 통계적 특성이 변함
- **발성 메커니즘**:
  - 유성음 (Voiced): 성대 진동, 주기적 파형 (모음)
  - 무성음 (Unvoiced): 공기 마찰, 비주기적 파형 (자음)

### 사람의 청각 특성을 반영한 표현

- **멜 스케일 변환**: $m(f) = 2595 \cdot \log_{10}(1 + \frac{f}{700})$
- **마스킹 현상**: 큰 소리가 가까운 주파수의 작은 소리를 가림
- **로그 스케일**: $L_{\text{dB}} = 20 \log_{10}(\frac{P}{P_0})$

### 음성 처리 파이프라인 구성 요소

1. **Wake Word Detection**: "Hey Siri", "OK Google" 등
2. **VAD (Voice Activity Detection)**: 말하기 시작/끝 시점 검출
3. **ASR (Automatic Speech Recognition)**: 음성을 텍스트로 변환
   - WER (Word Error Rate): $WER = \frac{S + D + I}{N} \times 100\%$ (S: 대체, D: 삭제, I: 삽입, N: 총 단어 수)
4. **NLU (Natural Language Understanding)**: 문맥과 의도 파악
5. **TTS (Text-to-Speech)**: 텍스트를 음성으로 합성

### 대표 응용: 자동 자막 & 회의 기록

- 오디오 스트림 추출 → 프레임 단위 분석 → ASR 디코딩 → 후처리 (문장부호/케이싱 복원) → 자막 싱크 → 화자 분리

### 대표 응용: 감정 분석 (Paralinguistics)

- 주요 특징: F0 (기초 주파수), 에너지, 스펙트럼 특징, 발화 속도

### 대표 응용: 화자 인식 (Speaker Recognition)

| 구분 | 설명 |
| --- | --- |
| Speaker Verification (화자 인증) | "이 사람이 맞는가?" (1:1) |
| Speaker Identification (화자 식별) | "누가 말했는가?" (1:N) |

- 주요 특징: Formant 구조, Voice Quality, 발화 습관, MFCC, i-vector, x-vector
- 현대적 접근: x-vector (DNN 임베딩), self-supervised models (wav2vec2, HuBERT)

### 멀티모달 음성 처리

- 음성 + 텍스트: 의도 및 감정 해석 강화
- 음성 + 영상: 립리딩, 표정, 시선
- 오디오 이벤트 인식: 환경 소리 + 말소리

### 음성 데이터 전처리 기법

#### Pre-emphasis (고주파 강조)

- 고주파 성분을 강조하여 스펙트럼 평탄화
- 필터 수식: $y[n] = x[n] - \alpha \cdot x[n-1]$ (α ≈ 0.95~0.97)

```python
def pre_emphasis(signal, alpha=0.95):
    return np.append(signal[0], signal[1:] - alpha * signal[:-1])
```

#### Noise Gate (노이즈 게이트)

- 임계값 이하의 신호 제거하여 배경 잡음 억제

```python
def noise_gate(signal, threshold=0.01, beta=0):
    gated = np.copy(signal)
    mask = np.abs(signal) < threshold
    gated[mask] = beta * signal[mask]
    return gated
```

#### Dynamic Range Compression (DRC)

- 큰 소리는 줄이고 작은 소리는 키워 전체 음량 균일화

#### Silence Trimming (묵음 제거)

```python
import librosa
y, sr = librosa.load("speech.wav")
y_trimmed, index = librosa.effects.trim(y, top_db=20, frame_length=512, hop_length=128)
```

### 음성 특징 시각화 실습

```python
import numpy as np
import matplotlib.pyplot as plt
import librosa
import librosa.display

AUDIO_PATH = "your_audio.wav"
SR_TARGET = 16000
N_FFT = 1024
HOP_LENGTH = 256
WIN_LENGTH = 1024
N_MELS = 80
N_MFCC = 13

def pre_emphasis(y, alpha=0.97):
    return np.append(y[0], y[1:] - alpha * y[:-1])

y, sr = librosa.load(AUDIO_PATH, sr=SR_TARGET)
y_emph = pre_emphasis(y, alpha=0.97)

# Waveform
plt.figure(figsize=(10, 3))
librosa.display.waveshow(y_emph, sr=sr)
plt.title("Waveform (pre-emphasized)")
plt.show()

# Spectrogram
S_complex = librosa.stft(y_emph, n_fft=N_FFT, hop_length=HOP_LENGTH, win_length=WIN_LENGTH, window="hann")
S_mag = np.abs(S_complex)
S_db = librosa.amplitude_to_db(S_mag, ref=np.max)
plt.figure(figsize=(10, 4))
librosa.display.specshow(S_db, sr=sr, hop_length=HOP_LENGTH, x_axis="time", y_axis="linear")
plt.title("Spectrogram (|STFT| in dB)")
plt.show()

# Mel-Spectrogram
M = librosa.feature.melspectrogram(y=y_emph, sr=sr, n_fft=N_FFT, hop_length=HOP_LENGTH, n_mels=N_MELS, power=2.0)
M_db = librosa.power_to_db(M, ref=np.max)
plt.figure(figsize=(10, 4))
librosa.display.specshow(M_db, sr=sr, hop_length=HOP_LENGTH, x_axis="time", y_axis="mel")
plt.title("Mel-Spectrogram (dB)")
plt.show()

# MFCC
MFCC = librosa.feature.mfcc(S=librosa.power_to_db(M), n_mfcc=N_MFCC)
plt.figure(figsize=(10, 4))
librosa.display.specshow(MFCC, x_axis="time", sr=sr, hop_length=HOP_LENGTH)
plt.title(f"MFCC (n_mfcc={N_MFCC})")
plt.show()
```

### 딥러닝과 음성 신호

| 모델 | 장점 | 단점 | 사용 사례 |
| --- | --- | --- | --- |
| RNN/LSTM | 순차 데이터 적합, 장거리 의존성 | 직렬 계산, 병렬화 어려움 | 프레임 단위 음향 특징 처리 |
| CNN | 지역적 패턴 포착, 병렬화 쉬움 | 전역 의존성은 깊이 필요 | Mel-spectrogram 이미지 처리 |
| Transformer | 병렬 처리, 전역 의존성 포착 | 시퀀스 길이 ↑ → O(T²) | End-to-End ASR, 긴 발화 |

### End-to-End ASR

- 전통적 접근: 특징 추출 → 음향 모델 → 발음 사전 → 언어 모델 (각 단계 별도 최적화)
- End-to-End: 단일 신경망으로 음성 → 텍스트 직접 매핑
- 장점: 통합 단일 모델, 발음 사전 불필요, 구현/배포 단순화, 오류 전파 최소화
